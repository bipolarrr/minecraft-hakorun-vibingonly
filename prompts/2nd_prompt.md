지금은 새 기능을 추가하는 단계가 아니라, 이미 생성된 코드에서 마지막 빨간 컴파일 에러 1개를 제거하는 단계다.

너는 시니어 Java/Paper 플러그인 디버거처럼 행동하라.

작업 원칙:
- 전체 리팩토링 금지
- 최소 수정 원칙
- 에러가 난 지점만 보지 말고 선언부/호출부/의존성 버전까지 같이 보라
- 에러를 없애기 위해 기능을 삭제하지 마라
- 내 코드 의도를 최대한 보존하라
- 수정 후 빌드가 통과될 가능성이 가장 높은 안을 우선 제시하라

반드시 해라:
1. 에러 메시지를 분해해서 “실제 문제 타입”을 먼저 판정
   - import 오류인지
   - 심볼 미해결인지
   - 타입 불일치인지
   - 메서드 시그니처 오류인지
   - 생성자 인자 불일치인지
   - static/non-static 문맥 문제인지
   - 버전 불일치인지
2. 그 다음 관련 코드와 pom.xml을 함께 보고 가장 유력한 원인을 하나로 좁혀라
3. 수정은 가능한 한 적은 파일만 건드려라
4. 최종적으로 수정된 코드 전체를 그대로 다시 써라

입력 자료:

[정확한 에러 메시지]
The method generator(ChunkGenerator) is ambiguous for the type WorldCreator

[문제 파일]
WorldManager.java

[pom.xml]
pom.xml을 참조하라.

원하는 답변 형식:
- 문제 타입
- 가장 유력한 원인
- 수정 포인트
- 수정 전/후 핵심 차이
- 최종 수정 코드
- 왜 이 수정이 가장 확실한지
